# 摆摊666APP 产品设计方案

---

## 一、产品定位

### 1.1 产品名称
**摆摊666**（别名：摆摊收款叫号助手）

### 1.2 目标用户
- 路边小吃摊主（鸡蛋灌饼、手抓饼、烤冷面、炸串等）
- 街边小餐饮个体户
- 早市、夜市摊位经营者

### 1.3 核心痛点与解决方案

| 痛点 | 解决方案 |
|------|----------|
| 忙时顾不上收款、找零、记单 | 顾客自助扫码下单+支付，无需摊主操作 |
| 排队混乱，容易错单、漏单 | 数字化取餐号系统，按号制作/叫号 |
| 无法一边制作一边处理订单 | 语音播报+APP实时推送，解放双手 |
| 无法统计当日营收 | 自动生成日报，订单数、营业额一目了然 |

### 1.4 产品形态
- **顾客端**：微信/支付宝小程序（扫码即用，无需下载）
- **摊主端**：Android/iOS APP（极简设计，单手操作）

### 1.5 核心价值主张
> **"边做边卖、边接单、边收款、叫号不用喊"**

## 二、功能模块

### 2.1 顾客端（小程序）

#### 2.1.1 扫码入口
- 顾客扫描摊位二维码，直接进入下单页面
- 无需注册、下载，微信/支付宝内完成

#### 2.1.2 语音下单
- 点击麦克风图标，说出需求
- 示例："我要一个鸡蛋灌饼，加肠加辣条，不要香菜"
- AI识别后转成文字，顾客可修改确认

#### 2.1.3 商品展示
- 摊位自主配置的商品菜单
- 支持规格选择（大小份、加料、加辣等）
- 价格实时显示

#### 2.1.4 订单确认与支付
- 展示完整订单：商品+规格+备注+金额
- 微信支付/支付宝支付
- 支付成功后生成唯一取餐号（如：A05）

#### 2.1.5 取餐进度
- 显示当前排队人数
- 显示预计等待时间（根据当前制作中的订单数估算）
- 叫号时推送通知（微信服务通知/支付宝消息）

#### 2.1.6 取餐核销
- 摊主标记"已完成"时，顾客端显示取餐提醒
- 顾客出示取餐码，摊主扫码核销（或摊主直接点击叫号）

---

### 2.2 摊主端（APP）

#### 2.2.1 摊主设置
- **店铺信息**：店名、头像、公告
- **商品管理**：添加/编辑/删除商品，设置价格、规格、库存
- **二维码生成**：生成顾客扫码下单的摊位二维码（可打印张贴）

#### 2.2.2 订单管理
- **实时订单列表**：按取餐号排序
- **订单卡片信息**：取餐号 + 商品 + 规格/备注 + 支付状态 + 状态进度
- **状态流转**：待接单 → 制作中 → 已完成 → 已取餐

#### 2.2.3 语音播报
- 新订单到达时，语音播放："新订单，A05号，鸡蛋灌饼加肠"
- 支持息屏/锁屏播报（后台服务）
- 可设置播报音量、播报内容详情/简略

#### 2.2.4 一键操作
- **接单**：确认接单，开始制作
- **完成**：点击"完成"，触发叫号
- **叫号**：屏幕显示+语音播放"A05号，请取餐"
- **核销**：顾客出示取餐码，扫码确认取餐

#### 2.2.5 收款与对账
- 微信/支付宝收款码合一
- 每笔订单支付状态实时显示（已支付/待支付）
- 收入自动到账，无需手动核对

#### 2.2.6 简易日报
- 今日订单数
- 今日营业额
- 热销商品排行
- 可选择日期查看历史数据

#### 2.2.7 消息通知
- 新订单提醒（声音+震动）
- 支付到账提醒
- 顾客取餐提醒

### 流程说明

1. **顾客扫码** → 进入小程序，展示摊位信息与商品菜单
2. **顾客点单** → 语音或手动选择商品，填写备注
3. **顾客支付** → 微信/支付宝完成支付
4. **系统生成取餐号** → 如 A05，发送给顾客
5. **摊主端收到订单** → 语音播报"新订单，A05号"
6. **摊主制作** → 按取餐号顺序制作
7. **摊主点击完成** → 触发叫号
8. **语音+屏幕叫号** → "A05号请取餐"
9. **顾客取餐** → 出示取餐码或直接报号确认
10. **订单完成** → 状态更新，流程结束

## 四、核心页面说明

### 4.1 顾客端页面

#### 页面一：首页/商品页
- 顶部显示摊位名称+公告
- 商品卡片简洁，大图+名称+价格
- 规格选项以胶囊按钮呈现，点击即选
- 底部悬浮购物车，方便随时结算
- 语音按钮突出，一键唤醒

#### 页面二：订单确认页
- 订单明细清晰，规格备注完整展示
- 金额自动汇总，一目了然
- 支付按钮大而醒目
- 底部快捷操作：补充备注 / 修改订单

#### 页面三：取餐等待页
- 取餐号大号加粗，醒目显示
- 排队人数和等待时间实时更新
- 订单详情可展开查看
- 叫号时微信服务通知推送

---

### 4.2 摊主端页面

#### 页面四：订单列表主页
- 顶部显示今日订单数+营业额汇总
- 订单卡片按取餐号顺序排列（倒序，最新单在前）

- 每个订单醒目展示：取餐号 + 商品 + 备注 + 支付状态
- 大按钮操作：接单 / 完成 / 叫号
- 单手可用，拇指区操作

#### 页面五：商品管理页
- 商品列表简洁，每行显示：名称 + 价格 + 库存状态 + 上下架状态
- 右滑快速编辑/删除
- 添加按钮悬浮在底部，单手可触达

#### 页面六：简易日报页
- 关键数据大号展示：订单数、营业额
- 热销商品排行，一目了然
- 订单高峰时段分布
- 支持查看历史数据

## 五、亮点与优势总结

### 5.1 核心亮点

| 亮点 | 说明 |
|------|------|
| **语音下单** | 顾客说一句话即可完成点单，解放双手 |
| **语音播报** | 摊主即使在忙其他事，也能第一时间知道新订单 |
| **取餐号系统** | 取代手写单据，彻底解决错单、漏单问题 |
| **一键叫号** | 屏幕显示+语音播放，嘈杂环境也不怕 |
| **自动对账** | 支付即到账，无需逐笔核对 |
| **极简设计** | 大字体、大按钮，单手操作，无广告 |

### 5.2 竞争优势

1. **零门槛接入**：无需营业执照，个人摊主即可使用
2. **低成本启动**：免费开通，仅支付通道手续费
3. **多平台支持**：微信小程序+支付宝小程序，全渠道覆盖
4. **离线可用**：网络不稳定时仍可缓存操作
5. **数据沉淀**：经营数据自动记录，辅助决策

### 5.3 用户价值

- **顾客**：扫码即用，不用排队，不用喊单，支付快捷，取餐有序
- **摊主**：专注制作，不用分心收款记单，订单不乱，收入清晰

### 5.4 商业模式

- **基础版**：免费（功能足够小摊使用）
- **高级版**：月费 ¥19.9（更多商品槽位、详细数据分析、去除广告等）
- **支付通道费**：按交易额收取（微信/支付宝标准费率）

## 六、技术架构

### 6.1 技术选型

| 端 | 技术栈 |
|----|--------|
| 摊主端APP | Flutter (Dart) |
| 顾客端小程序 | 微信小程序/支付宝小程序 |
| 后端服务 | Node.js + Express + MongoDB |
| 实时通信 | WebSocket |
| 语音合成 | TTS (Text-to-Speech) |
| 语音识别 | 微信/支付宝小程序原生语音 |

### 6.2 数据模型

#### 商品 (Product)
```
{
  id: string,
  name: string,
  price: number,
  image: string,
  category: string,
  specs: [{ name: string, options: [{ label: string, price: number }] }],
  stock: number,
  status: 'active' | 'inactive',
  createdAt: Date
}
```

#### 订单 (Order)
```
{
  id: string,
  orderNumber: string,        // 取餐号，如 A05
  items: [{
    productId: string,
    name: string,
    price: number,
    specs: string,
    quantity: number
  }],
  totalAmount: number,
  status: 'pending' | 'cooking' | 'completed' | 'taken',
  paymentStatus: 'paid' | 'unpaid',
  paymentMethod: 'wechat' | 'alipay',
  remarks: string,
  customerId: string,
  createdAt: Date,
  completedAt: Date,
  takenAt: Date
}
```

#### 摊位 (Stall)
```
{
  id: string,
  name: string,
  ownerId: string,
  logo: string,
  notice: string,
  qrcode: string,
  products: Product[],
  stats: {
    todayOrders: number,
    todayRevenue: number
  }
}
```

---

**结语**：摆摊666用数字化手段解决线下小吃摊的全流程痛点，让"边做边卖"真正成为可能，让每一位摊主都能轻松当老板。
